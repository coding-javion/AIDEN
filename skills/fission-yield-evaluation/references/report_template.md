# 裂变产额审计报告模板

最终回答应使用这个结构。可以根据任务规模压缩，但不要省略关键状态和证据。

## 总体判定

- `PASS`：未发现违反硬约束的问题，经验系统学也大体合理。
- `WARN`：存在可疑趋势或元数据不足，但没有硬错误。
- `FAIL`：存在明确硬约束违规。
- `BLOCKER`：关键元数据缺失，无法完成用户请求的核心判断。

写一句简短结论，说明最重要的依据。

## 元数据门槛

用清单列出：

- 裂变体系或复合核；
- 入射道；
- 入射能量或激发能；
- 产额阶段；
- 产额类型；
- 归一化；
- 完整性；
- 不确定度/协方差信息。

缺失项写 `INSUFFICIENT_METADATA`，不要补造。

## 检查清单

建议表格列：

```markdown
| 规则 | 状态 | 证据 | 说明 |
|---|---|---|---|
| NUM-001 | PASS | 产额列 1240 行均有限且非负 | 使用 scripts/fpy_basic_audit.py |
| NUM-003 | NA | 数据说明为累积产额 | 不对累积产额求和判归一化 |
```

每条 `FAIL` 或 `WARN` 必须能复查：

- 写出文件路径、列名、行号或核素；
- 写出数值，如总和、峰位、最小值、最大值、最小特征值；
- 写出适用域，例如“低能锕系完整质量产额”。

## 主要发现

按严重性排序：

1. `FAIL`；
2. `BLOCKER`；
3. `WARN`；
4. `INFO`。

每条发现使用格式：

```markdown
- **规则 ID：一句话问题。** 证据：... 影响：... 建议：...
```

## 保守解释

说明哪些规则没有应用，以及原因：

- 元数据不足；
- 数据是部分测量集合；
- 产额类型是累积而非独立；
- 体系/能区不属于低能锕系；
- 缺少参考库或规范不匹配。

## 后续建议

只给与审计直接相关的建议，例如：

- 补充产额类型、归一化和前/后中子约定；
- 提供协方差或不确定度；
- 与指定评价库做同规范比较；
- 对异常质量链或核素做源数据回查。
